# 차일디 목표관리 시스템 - 프로젝트 개요

## 📋 프로젝트 소개

**차일디 목표관리 시스템**은 조직의 성과 기반 업무 관리 및 리소스 모니터링을 위한 웹 기반 통합 플랫폼입니다. OKR(Objectives and Key Results) 방법론을 기반으로 목표 설정, 진척도 관리, 직무기술서 및 R&R(Role & Responsibility) 관리를 통합하여 조직의 투명성과 효율성을 극대화합니다.

---

## 🎯 시스템 목적

### 1. 목표 중심의 조직 운영
- **OKR 기반 목표 관리**: 월별/분기별/연간 단위로 명확한 목표 설정 및 추적
- **투명한 성과 가시화**: 모든 구성원의 목표와 진척도를 실시간으로 공유
- **상향식 목표 합의**: 구성원이 목표를 제안하고 리더가 승인하는 협업적 프로세스

### 2. 업무 명확화 및 책임 소재 확립
- **직무기술서 관리**: 각 구성원의 전문 역량과 수행 가능 업무 범위 명시
- **R&R 정의**: 담당 업무, 프로젝트별 과업, 리소스 투입 시간(M/D, M/M) 구체화
- **업무 중복 방지**: 명확한 역할 정의로 업무 효율성 향상

### 3. 데이터 기반 의사결정
- **실시간 대시보드**: 조직 전체의 목표 달성률 및 진척도 한눈에 파악
- **요청 관리 시스템**: 모든 변경 사항에 대한 승인 프로세스 및 이력 관리
- **리소스 모니터링**: 구성원별 업무 부하 및 시간 배분 추적

---

## 💡 핵심 의도

### 조직 문화 혁신
1. **투명성 강화**: 모든 목표와 업무가 가시화되어 조직 내 정보 비대칭 해소
2. **자율성 부여**: 구성원이 스스로 목표를 설정하고 관리하는 주도적 업무 환경
3. **책임감 향상**: 명확한 목표와 역할 정의로 개인의 기여도 명확화
4. **협업 촉진**: 리더와 구성원 간 지속적인 피드백과 코멘트 교환

### 성과 관리 체계화
1. **정량적 평가 기반 마련**: Key Results의 진척률로 객관적 성과 측정
2. **주기적 리뷰 체계**: 월/분기/연 단위 목표 설정 및 회고 사이클 구축
3. **변경 이력 관리**: 모든 수정 요청과 승인 과정 기록 및 추적
4. **데이터 축적**: 장기적 성과 데이터 분석을 위한 기반 구축

---

## 🚀 기대 효과

### 조직 차원
- **전략 실행력 향상**: 조직 목표가 개인 목표로 연결되어 전략 실행 가속화
- **의사결정 속도 개선**: 실시간 데이터 기반으로 빠른 의사결정 가능
- **리소스 최적화**: 구성원별 업무 부하 가시화로 효율적 인력 배치
- **조직 학습 촉진**: 목표 달성 과정의 데이터 축적으로 조직 학습 강화

### 리더 차원
- **팀 성과 모니터링**: 팀원들의 목표 진척도를 실시간으로 파악
- **효과적인 피드백**: 구체적인 데이터 기반 코칭 및 피드백 제공
- **승인 프로세스 간소화**: 체계적인 요청 관리로 의사결정 효율성 증대
- **공정한 평가 기반**: 객관적 데이터로 공정한 성과 평가 가능

### 구성원 차원
- **목표 명확성**: 자신의 목표와 기대 성과가 명확하게 정의됨
- **성장 가시화**: 진척도 추적으로 자신의 성장과 기여도 확인
- **업무 자율성**: 스스로 목표를 설정하고 관리하는 주도적 업무 방식
- **경력 개발**: 직무기술서를 통한 역량 관리 및 경력 개발 계획 수립

## 🔧 주요 기능

### 1. 대시보드
**목적**: 조직 전체의 목표 달성 현황을 한눈에 파악

**기능**:
- 월별/분기별/연간 목표 현황 조회
- 구성원별 OKR 및 Key Results 진척률 표시
- 기간별 필터링 및 검색 기능
- 실시간 데이터 동기화

**사용자**: 전체 (리더/멤버)

---

### 2. 내 목표 관리
**목적**: 개인의 목표 진척도를 관리하고 업데이트

**기능**:
- 승인된 OKR 및 Key Results 조회
- Key Results 진척률 실시간 업데이트
- 진척률 변경 시 수정 요청 제출
- 거부된 요청에 대한 리더 코멘트 확인
- 월별/분기별/연간 기간 선택

**워크플로우**:
1. 승인된 목표 확인
2. Key Results 진척률 입력
3. 변경 사항 저장 또는 수정 요청 제출
4. 리더 승인 대기
5. 승인 시 자동 반영, 거부 시 코멘트 확인 후 재제출

**사용자**: 전체 (리더/멤버)

---

### 3. 목표 설정 및 합의
**목적**: 새로운 목표를 설정하고 리더의 승인을 요청

**기능**:
- 새로운 OKR 작성
- Key Results 추가/수정/삭제
- 목표 설정 시 코멘트 첨부
- 승인 요청 제출
- 요청 취소 및 재작성

**OKR 작성 가이드**:
- **Objective**: 도전적이고 영감을 주는 정성적 목표
- **Key Results**: 측정 가능한 정량적 성과 지표 (3-5개 권장)
- 진척률은 0-100% 범위로 관리

**워크플로우**:
1. 기간 선택 (월별/분기별/연간)
2. OKR 작성
3. Key Results 추가 (최소 1개)
4. 코멘트 작성 (선택)
5. 승인 요청 제출
6. 리더 검토 및 승인/거부
7. 승인 시 '합의 완료' 상태로 전환

**사용자**: 전체 (리더/멤버)

---

### 4. 직무기술 / R&R
**목적**: 구성원의 전문 역량과 담당 업무를 명확히 정의

**기능**:
- **직무기술서 작성**: 보유 역량, 사용 가능 툴, 숙련도, 수행 가능 업무 범위
- **R&R 작성**: 담당 업무, 프로젝트별 과업, 소요 시간(M/D, M/M), 대시보드 링크
- 합의 요청 및 수정 요청
- 리더 코멘트 확인
- 거부 시 재작성 및 재제출

**작성 가이드**:
- **직무기술**: 
  - 사용 가능한 툴과 숙련도 명시
  - 구체적인 직무 영역 기술
  - 예: "Figma를 활용한 컴포넌트 기반 디자인 시스템 구축 가능"
  
- **R&R**:
  - 현재 담당 업무와 프로젝트별 상세 과업
  - 실제 소요 시간을 M/D 또는 M/M 단위로 산정
  - 참고 대시보드나 리포트 링크 첨부
  - 예: "주간 매체 효율 최적화에 주 2일(월 0.4 M/M) 투입"

**워크플로우**:
1. 직무기술 및 R&R 작성
2. 코멘트 첨부 (선택)
3. 합의 요청 제출
4. 리더 검토 및 승인/거부
5. 거부 시 리더 코멘트 확인 후 수정
6. 승인 시 '합의 완료' 상태로 전환

**사용자**: 전체 (리더/멤버)

---

### 5. 요청 관리 (관리자 전용)
**목적**: 구성원의 모든 요청을 검토하고 승인/거부

**기능**:
- OKR 신규 수립 요청 검토
- OKR/KR 수정 요청 검토
- 진척률 변경 요청 검토
- 직무기술/R&R 등록 및 수정 요청 검토
- 상세 내용 전후 비교 (AS-IS / TO-BE)
- 승인/거부 처리
- 거부 시 코멘트 작성
- 요청일시 확인
- 승인 취소 기능

**요청 유형**:
1. **신규 수립**: 새로운 OKR 작성
2. **OKR 수정**: 목표 내용 변경
3. **KR 추가/수정/삭제**: Key Results 변경
4. **진척률 조정**: Key Results 진척률 업데이트
5. **직무기술 등록/수정**: 직무기술서 작성 또는 변경
6. **R&R 등록/수정**: R&R 작성 또는 변경

**워크플로우**:
1. 요청 목록 조회 (기간별 필터링)
2. 요청 상세 내용 확인
3. 전후 비교 검토 (수정 요청의 경우)
4. 코멘트 확인 (구성원이 작성한 요청 사유)
5. 승인 또는 거부 결정
6. 거부 시 거부 사유 코멘트 작성
7. 구성원에게 결과 통보 (자동)

**사용자**: 리더만

---

### 6. 구성원 관리 (관리자 전용)
**목적**: 조직 구성원 정보를 관리

**기능**:
- 구성원 목록 조회
- 신규 구성원 추가
- 구성원 정보 수정 (이름, 이메일, 직무, 직책, 팀)
- 구성원 삭제
- 일괄 저장 기능
- 비밀번호 관리

**관리 항목**:
- 이름
- 이메일
- 직무
- 직책 (리더/멤버)
- 팀
- 사용자 ID
- 비밀번호

**사용자**: 리더만

---

### 7. OKR 가이드
**목적**: OKR 작성 방법 및 시스템 사용법 안내

**기능**:
- OKR 개념 및 작성 원칙 설명
- 좋은 OKR 예시 제공
- 시스템 사용 가이드
- 작성 시기 및 주기 안내
- FAQ 및 팁 제공

**주요 내용**:
- **OKR이란?**: Objectives and Key Results 방법론 소개
- **작성 원칙**: 
  - Objective: 도전적, 영감적, 정성적
  - Key Results: 측정 가능, 구체적, 시간 제한
- **작성 예시**: 실제 사례 기반 예시
- **시스템 사용법**: 단계별 사용 가이드
- **작성 시기**: 월말/분기말/연말 작성 권장

**사용자**: 전체 (리더/멤버)

## 🏗️ 시스템 아키텍처

### 기술 스택
- **Frontend**: HTML5, CSS3 (Tailwind CSS), Vanilla JavaScript
- **Backend**: Baserow (No-code Database Platform)
- **Deployment**: GitHub Pages (Static Hosting)
- **API**: Baserow REST API
- **Authentication**: Session-based (localStorage)

### 데이터베이스 구조 (Baserow)

#### 1. divisions (본부)
- `id`: 고유 ID
- `name`: 본부명

#### 2. teams (팀)
- `id`: 고유 ID
- `name`: 팀명
- `division`: 소속 본부

#### 3. members (구성원)
- `id`: 고유 ID
- `user_id`: 사용자 ID (로그인용)
- `password`: 비밀번호
- `name`: 이름
- `email`: 이메일
- `division`: 소속 본부
- `team`: 소속 팀
- `position`: 직책 (리더/멤버)
- `job`: 직무

#### 4. goals (목표)
- `id`: 고유 ID
- `user_id`: 작성자 ID
- `period_type`: 기간 유형 (monthly/quarterly/yearly)
- `period_value`: 기간 값 (2026-04, 2026-Q1, 2026)
- `OKR`: 목표 내용
- `status`: 상태 (승인 대기중/합의 완료)
- `request_type`: 요청 유형 (신규 수립, OKR 수정 등)
- `comment`: 요청 코멘트
- `reject_comment`: 거부 코멘트
- `is_processed`: 승인 여부
- `temp_text`: 임시 OKR 텍스트 (수정 요청 시)
- `temp_kr`: 임시 Key Results (JSON)
- `request_date`: 요청일시

#### 5. key_results (핵심 결과)
- `id`: 고유 ID
- `goal_id`: 목표 ID (외래키)
- `kr_id`: Key Result ID
- `OKR`: 목표 내용 (중복 저장)
- `KR`: Key Result 내용
- `progress`: 진척률 (0-100)

#### 6. rnr (직무기술 및 R&R)
- `id`: 고유 ID
- `user_id`: 작성자 ID
- `name`: 이름
- `team`: 팀
- `position`: 직책
- `job`: 직무기술
- `rnr`: R&R 내용
- `status`: 상태 (승인 대기중/합의 완료)
- `request_type`: 요청 유형
- `comment`: 요청 코멘트
- `reject_comment`: 거부 코멘트
- `temp_content`: 임시 내용 (JSON)
- `request_date`: 요청일시

### 주요 설계 패턴

#### 1. 승인 워크플로우
```
구성원 작성 → 요청 제출 → 리더 검토 → 승인/거부
                                    ↓
                              승인: 데이터 확정
                              거부: 코멘트 전달 → 재작성
```

#### 2. 임시 데이터 관리
- 수정 요청 시 원본 데이터 유지
- `temp_*` 필드에 변경 사항 저장
- 승인 시 임시 데이터를 원본으로 이동
- 거부 시 임시 데이터 삭제

#### 3. 로컬 우선 생성
- 신규 OKR은 로컬에서 먼저 생성 (`isLocalOnly: true`)
- 승인 요청 시 Baserow에 저장
- 취소 시 로컬 데이터만 삭제

#### 4. 세션 관리
- localStorage 기반 세션 (24시간 유효)
- 자동 로그인 복원
- 권한 기반 메뉴 표시

## 🔄 향후 개선 방향

### 단기 (1-3개월)
1. **알림 시스템**: 요청 승인/거부 시 이메일 또는 푸시 알림
2. **검색 기능**: 목표 및 구성원 검색 강화
3. **엑셀 내보내기**: 데이터 분석을 위한 엑셀 다운로드
4. **비밀번호 암호화**: 보안 강화

### 중기 (3-6개월)
1. **팀 대시보드**: 팀 단위 성과 집계 및 시각화
2. **목표 연결**: 조직 목표 → 팀 목표 → 개인 목표 연결
3. **진척도 그래프**: 시간에 따른 진척도 변화 시각화
4. **코멘트 스레드**: 요청별 다중 코멘트 및 토론 기능

### 장기 (6개월 이상)
1. **AI 추천**: 과거 데이터 기반 목표 추천
2. **성과 리포트**: 자동 생성되는 월간/분기별 성과 리포트
3. **모바일 앱**: 네이티브 모바일 앱 개발
4. **통합 연동**: Slack, Teams 등 협업 툴 연동
5. **평가 시스템**: OKR 기반 성과 평가 자동화
